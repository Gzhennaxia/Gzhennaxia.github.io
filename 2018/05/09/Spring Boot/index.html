  • spring boot的pom文件中有一个parent标签，用与继承父项目的配置

  • spring boot内嵌tomcat，可以以jar包的形式独立运行

  • 入口类XXXApplication上的注解 @SpringBootApplication等价于@SpringBootConfiguration @EnableAutoConfiguration @ComponentScan三个注解的组合

  • spring boot配置文件application.properties或者application.yml,spring推荐使用YAML

    • Yaml中使用缩进表示层级关系缩进只能用空格，空格数目不重要，只要相同层级的元素左侧对齐即可，不要用tab，否者会出现诡异的问题

    • Yaml大小写敏感（区分大小写）

    • 每个key的冒号后面都一定要加一个空格

    • 可以在类上使用@ConfigurationProperties来读取配置文件中自定义的内容

  • 提供@RestController注解，方法上不用再加@ResponseBody注解，默认返回的就是json数据，更激进的支持前后端分离。

打包方式

jar包方式之一：maven命令行打包

  1. cd到项目的target同级目录
  2. 执行打包命令 mvn clean package
  3. 启动命令 java -jar 包名.jar

jar包方式之二：idea插件打包

在IDEA右侧Maven Projects栏双击package等待Build Success即可

  1. 为什么我们需要 spring-boot-maven-plugin?

    spring-boot-maven-plugin 提供了一些像 jar 一样打包或者运行应用程序的命令。

    • spring-boot:run 运行你的 SpringBoot应用程序。
    • spring-boot:repackage 重新打包你的 jar 包或者是 war 包使其可执行
    • spring-boot:start 和 spring-boot:stop 管理 Spring Boot 应用程序的生命周期（也可以说是为了集成测试）。

  2. @RequestMapping 和 @GetMapping 的不同之处在哪里？

    @GetMapping是@RequestMapping(method = RequestMethod.GET)的缩写
