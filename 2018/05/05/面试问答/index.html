框架

  1. spring如何做到轻量级的（核心功能）？

    1. 独立运行的spring项目

      Spring Boot可以以jar包的形式进行独立的运行

    2. 内嵌servlet容器

      内嵌容器，使得我们直接执行项目的主方法main函数，就可以实现项目的快速运行；

    3. 提供starter简化Manen配置

      Spring Boot提供了一系列的starter pom用来简化我们的Maven依赖

    4. 自动配置Spring

      Spring Boot会根据我们项目中类路径的jar包/类，为jar包的类进行自动配置Bean，这样一来就大大的简化了我们的配置。

    5. 无代码生成和XML配置

      Spring Boot神奇的地方不是借助于代码生成来实现的，而是通过条件注解的方式来实现的。

  2. 你们的项目有做什么优化，性能测试?

    主要从页面、数据库、服务器。。。等方面进行优化

    1. 页面

      1. 压缩js/css文件（去空格，缩短变量名）
      2. 页面静态化（不需要查询数据库）

    2. 数据库

      1. 分表

        纵向分表：将操作不频繁的表分离出来

        横向分表：可以按两个维度进行分表：按时间分表，按用户分表

      2. 数据库缓存

      3. sql语句优化

        尽量不使用联查

        不要将表中数据全部查询

    3. 服务器

      1. 负载均衡

        代理服务器分配请求

        请求的分配方式：轮询【默认】，权重【能力大，责任就大】，ip_hash【根据ip地址来进行分配】
